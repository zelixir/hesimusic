# 开发计划总览

本项目旨在开发一个基于 Android 的本地音乐播放器，目标是替代天天动听。
开发过程分为 5 个阶段，每个阶段完成后需进行验收。

## 阶段一：基础架构与数据层 (当前阶段)
- [ ] **目标**: 搭建项目骨架，实现数据库设计，完成核心的文件扫描与 CUE 解析功能。
- [ ] **计划文件**: `plan_phase_1.md`
- [ ] **状态**: 未开始
- [ ] **验收方式**: 
    - App 能成功运行并申请 "所有文件访问权限"。
    - 点击扫描按钮能遍历手机存储。
    - 数据库中能正确看到扫描到的歌曲（包括 CUE 分轨）。
    - 能够识别并正确显示中文/日文编码的元数据。

## 阶段二：核心播放功能
- [ ] **目标**: 实现基于 ExoPlayer 的音频播放引擎，支持后台播放与通知栏控制。
- [ ] **计划文件**: `plan_phase_2.md`
- [ ] **状态**: 未开始
- [ ] **验收方式**:
    - 点击列表中的歌曲能正常播放。
    - CUE 分轨能正确从指定时间开始播放，并在结束时间停止或切歌。
    - App 切到后台音乐不中断。
    - 通知栏能显示当前歌曲信息并控制暂停/下一曲。

## 阶段三：UI 交互与列表视图
- [ ] **目标**: 完成主页面的 UI 开发，包括歌曲、歌手、专辑、文件夹四个 Tab 的展示与交互。
- [ ] **计划文件**: `plan_phase_3.md`
- [ ] **状态**: 未开始
- [ ] **验收方式**:
    - 首页四个 Tab 切换流畅。
    - 列表能正确显示数据，支持首字母索引快速定位。
    - 文件夹视图能正确层级跳转。
    - 列表项能正确反映当前播放状态（高亮）。

## 阶段四：播放详情与高级功能
- [ ] **目标**: 开发正在播放页面，实现播放队列管理、睡眠模式、均衡器等高级功能。
- [ ] **计划文件**: `plan_phase_4.md`
- [ ] **状态**: 未开始
- [ ] **验收方式**:
    - 正在播放页能显示封面、进度条，并能拖动进度。
    - 播放队列能查看、删除、清空。
    - 睡眠模式倒计时结束后能自动停止播放。
    - 均衡器调节能实时改变音效。

## 阶段五：优化与发布
- [ ] **目标**: 性能优化，UI 细节打磨，Bug 修复，准备发布。
- [ ] **计划文件**: `plan_phase_5.md`
- [ ] **状态**: 未开始
- [ ] **验收方式**:
    - 扫描 1000+ 首歌无卡顿。
    - 内存占用稳定，无内存泄漏。
    - 各种异常格式文件测试通过。
